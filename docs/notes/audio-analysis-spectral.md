# スペクトル・時間周波数・知覚特徴量

---

## 1. FFT / DFT

**何が分かるか**  
信号の周波数成分（振幅スペクトル・位相スペクトル・パワースペクトル）。スペクトル解析の根幹。

**計算式**

```
X[k] = Σ_{n=0}^{N-1} x[n] · exp(-j · 2π · k · n / N)
```

FFT はクーリー-テューキー法により O(N²) → O(N log N) に高速化。

**実装**

```python
from scipy.fft import rfft, rfftfreq
import numpy as np

N = len(signal)
window  = np.hanning(N)
X       = rfft(signal * window)
freqs   = rfftfreq(N, 1 / sample_rate)

magnitude = np.abs(X)
phase     = np.angle(X)
power     = magnitude ** 2
mag_db    = 20 * np.log10(np.maximum(magnitude / magnitude.max(), 1e-6))
```

### 窓関数の選択

| 窓関数 | 特徴 | 用途 |
|---|---|---|
| Hann | バランス良好 | 一般音楽解析 |
| Hamming | サイドローブやや大 | 音声処理 |
| Blackman | サイドローブ小・主弁幅広 | 高精度スペクトル推定 |
| Kaiser | パラメータで形状調整可 | 柔軟な用途 |
| Flat-top | 振幅精度最高 | キャリブレーション |

## 2. スペクトル特徴量

### スペクトル重心（Spectral Centroid）

**何が分かるか**: スペクトルエネルギーの「重心周波数」。音の明るさ（Brightness）に対応する。

```
SC = Σ(f[k] · |X[k]|) / Σ|X[k]|
```

```python
def spectral_centroid(magnitude, freqs):
    return np.sum(freqs * magnitude) / (np.sum(magnitude) + 1e-10)
```

### スペクトル帯域幅（Spectral Bandwidth）

**何が分かるか**: スペクトルの広がり。音色の豊かさ・複雑さに相関する。

```
SBW = sqrt( Σ((f[k] - SC)² · |X[k]|) / Σ|X[k]| )
```

### スペクトルフラックス（Spectral Flux）

**何が分かるか**: 連続する2フレーム間のスペクトル変化量。オンセット検出の主要特徴量。

```
SF = Σ (|X_t[k]| - |X_{t-1}[k]|)²
```

正方向のみ取る（増加だけ）と、オンセット検出により有効。

### スペクトルロールオフ（Spectral Rolloff）

**何が分かるか**: 全スペクトルエネルギーの指定割合（通常 85%）が含まれる周波数。高周波成分の存在判定。

### スペクトルフラットネス（Spectral Flatness）

**何が分かるか**: 純音（トーン）かノイズかを示す指標（Wienerエントロピー）。0 ≈ 純音、1 ≈ 白色雑音。

```
SF = (Π|X[k]|)^(1/N) / ((1/N) · Σ|X[k]|)   （幾何平均 / 算術平均）
```

### スペクトルコントラスト（Spectral Contrast）

**何が分かるか**: 各周波数帯域でのピーク成分と谷成分のエネルギー差。倍音構造・和音の識別に有効。

## 3. STFT（短時間フーリエ変換）とスペクトログラム

**何が分かるか**  
時間と周波数の両方の情報を持つスペクトログラム。音楽解析の最も基本的な時間-周波数表現。

```
STFT(t, f) = Σ_n x[n] · w[n - t] · exp(-j · 2π · f · n)
```

時間分解能と周波数分解能はトレードオフ（ハイゼンベルクの不確定性原理と類似）。

**実装**

```python
from scipy.signal import spectrogram

f, t, Sxx = spectrogram(signal, fs=sample_rate,
                         nperseg=2048, noverlap=1792,
                         window='hann')
Sxx_db = 10 * np.log10(np.maximum(Sxx, 1e-12))
```

**パラメータ選択指針**

| 用途 | nperseg | noverlap |
|---|---|---|
| 音楽解析（一般） | 2048 | 1536 |
| 打楽器・アタック | 512〜1024 | 3/4 |
| 低音ピッチ推定 | 4096〜8192 | 3/4 |
| リアルタイム | 512 | 1/2〜3/4 |

## 4. Mel スペクトログラム

**何が分かるか**  
人間の聴覚特性（対数的周波数知覚）に合わせた Mel 尺度のスペクトログラム。機械学習・音声認識の標準的な入力表現。

**Mel 変換式**

```
mel = 2595 · log10(1 + f / 700)
```

STFT のパワースペクトルに Mel フィルタバンク（三角形フィルタ群）を適用して得られる。

**STFT / Mel スペクトログラム / CQT の比較**

| 項目 | STFT | Mel スペクトログラム | CQT |
|---|---|---|---|
| 周波数スケール | 線形 | 対数（Mel尺度） | 対数（音楽的） |
| 低音域分解能 | 低い | 中 | 高い |
| 音楽解析適性 | 中 | 高い（ML向き） | 非常に高い |
| 計算コスト | 低 | 低〜中 | やや高い |

## 5. CQT（Constant-Q Transform、定Q変換）

**何が分かるか**  
周波数ビンが対数間隔のスペクトログラム。1半音 = 1ビン。音楽の音階・和声構造の解析に最適。

**Q値の定義**

```
Q = f_center / Δf = 1 / (2^(1/B) - 1)   （B: 1オクターブあたりのビン数）
```

低音域では窓長が長くなり（高周波数分解能）、高音域では短くなる（高時間分解能）。

---

## 6. ウェーブレット変換（CWT）

**何が分かるか**  
マルチスケールでの時間-周波数解析。非定常信号・過渡現象に有効。CQT と類似した対数スケール特性を持つ。

代表的なマザーウェーブレット：
- **Morlet** — 音楽解析に適する（正弦波 × ガウシアン）
- **Mexican hat** — エッジ・トランジェント検出
- **Daubechies（db4, db8）** — 多解像度解析

---

## 7. ガボール変換（Gabor Transform）

ガウス窓を使った STFT の特殊形。時間-周波数不確定性の観点で最適な窓関数。STFT の理論的基礎。

---

## 8. Wigner-Ville 分布（WVD）

**何が分かるか**  
理論的に最高の時間-周波数分解能。  
**ただし** 複数成分信号では交差項（クロスターム）が発生してノイズ状になる問題がある。  
単一成分の瞬時周波数追跡には有用。疑似WVD（PWVD）で短い窓を使うとクロスタームを抑制できる。

---

## 9. EMD / HHT（経験的モード分解 / ヒルベルト-黄変換）

**何が分かるか**  
非定常・非線形信号を固有モード関数（IMF: Intrinsic Mode Function）に分解。ピッチ変動・ビブラートの解析に有効。フーリエ変換と異なり、信号の局所的な特性を捉えられる。

```python
# pip install emd-signal
import emd
imfs = emd.sift.sift(signal)
IP, IF, IA = emd.spectra.frequency_transform(imfs, sample_rate, 'hilbert')
# IP: 瞬時位相, IF: 瞬時周波数, IA: 瞬時振幅
```

## 10. MFCC（Mel 周波数ケプストラム係数）

**何が分かるか**  
人間の聴覚特性を模倣した音色の「形状」。音声認識・話者識別・楽器分類・ジャンル分類の標準特徴量。  
シンセサイザーの音色識別・分類にも直接応用できる。

**計算パイプライン**

```
プリエンファシス → フレーム分割 + 窓関数 → FFT → パワースペクトル
  → Mel フィルタバンク（26バンク） → 対数変換 → DCT → 係数選択（通常 C1〜C13）
```

**係数の解釈**

| 係数番号 | 情報内容 |
|---|---|
| C0 | 全体エネルギー（通常除外） |
| C1〜C3 | スペクトルの大まかな形状 |
| C4〜C8 | 中程度の細部 |
| C9〜C13 | 高周波の微細構造 |

**デルタ特徴量**: 時間方向の差分（速度・加速度）を付加することで、音の「変化の仕方」も表現できる。

**LPC との関係**: LPC から得たスペクトルに DCT を適用したものが LPCC。MFCC は Mel フィルタバンク経由なので人間の聴覚により近い。

---

## 11. LPC（線形予測符号化）

**何が分かるか**  
音声の声道（フィルタ）モデルのパラメータ。フォルマント推定、音声合成・圧縮の基礎。シンセサイザーのスペクトル包絡推定に応用できる。

```
x[n] ≈ -Σ_{k=1}^{p} a[k] · x[n-k] + G · u[n]
```

Levinson-Durbin アルゴリズムで自己相関行列から係数を効率的に推定する。

---

## 12. クロマ特徴量（Chroma Features）

**何が分かるか**  
12 音階（C, C#, D...B）それぞれへのエネルギー分布。調性（キー）判定、コード認識、楽曲の調和的類似性の比較に最適。オクターブ不変（どのオクターブも同じ音名に折り畳まれる）。

**バリエーション**

| 名称 | 特徴 |
|---|---|
| Chroma-STFT | 基本形。計算が速い |
| Chroma-CQT | CQT ベース。精度が高い |
| CENS | エネルギー正規化 + 量子化 + 平滑化。楽曲検索向き |

---

## 13. Tonnetz（トーンネット特徴量）

**何が分かるか**  
和声・調性の 6 次元表現（純正律空間でのピッチクラスのマッピング）。音楽理論に基づいた和声解析に特有。  
完全5度・長3度・短3度の軸でピッチ空間を表現する。
