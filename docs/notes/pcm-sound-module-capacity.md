# PCM音源モジュールの容量・仕様 深掘り調査

音源モジュール・MIDI音源・ハードウェア音源に絞り、歴代の代表的なPCM音源のROM容量・サンプリング仕様・1音色あたりのメモリ割り当てなどを時代順に調査・整理した。

---

## 凡例・注意事項

- **実容量**：物理ROMに格納されたバイト数
- **実効容量**：圧縮展開後の相当バイト数（音質的な情報量）
- 「推定」と記した値は、公開情報から合理的に算出した推定値
- サンプラー（Akai, E-mu Emulator 等）はユーザーRAMが主体のため対象外とした

## 1. PCM黎明期（1987〜1989年）

### Roland D-50（1987年）

| 項目 | 内容 |
|---|---|
| 総波形ROM容量 | **512KB**（4Mbit、2チップ構成） |
| サンプリング周波数 | **約33,375 Hz**（独自クロック） |
| ビット深度 | **16bit**（2チップ並列構成） |
| ステレオ/モノラル | モノラル |
| 総波形数 | **100種**（アタック系PCM 47 + ループPCM 29 + スキャンニングループ 24） |
| 総音色数 | 64プリセット（拡張カードで+64） |
| 最大ポリフォニー | 16声 |
| データ圧縮 | なし（16bit 線形PCM） |

**特記：** D-50は「アタック継ぎ足し方式（LA合成）」を採用。アタック部のみPCMサンプルを再生し、サステイン部はデジタルオシレーター（のこぎり波・矩形波）で代替することで、わずか512KBで100種の波形を実現した。1音色は最大2パーシャル（アタックPCM + ループPCM）で構成。

| 推定値 | 計算 |
|---|---|
| 1サンプル平均容量 | 512KB ÷ 100 ≈ **約5KB** |
| 各サンプルの長さ（サンプル点数） | 約2,048〜16,384サンプル点（マニュアル記載） |
| **各サンプルの長さ（時間）** | 2,048 ÷ 33,375 ≈ **61ms** 〜 16,384 ÷ 33,375 ≈ **491ms** |
| 平均サンプル長（時間） | 512KB ÷ 2（16bit） ÷ 33,375Hz ÷ 100種 ≈ **約77ms** |
| 1音色あたりのメモリ | PCM部約5〜10KB（2パーシャル） |

### KORG M1（1988年）

| 項目 | 内容 |
|---|---|
| 総波形ROM容量 | **4MB**（M1R拡張キットで8MB） |
| サンプリング周波数 | **約31.25 kHz**（Korg独自クロック） |
| ビット深度 | **16bit** |
| ステレオ/モノラル | モノラル（一部ステレオあり） |
| 総波形数 | **100マルチサウンド + 44ドラムサウンド = 144種** |
| 総音色数 | プログラム100、コンビネーション100 |
| 最大ポリフォニー | 16声 |
| データ圧縮 | なし |

| 推定値 | 計算 |
|---|---|
| 1波形平均容量 | 4MB ÷ 144 ≈ **約28KB** |
| **1波形平均長（時間）** | 4MB ÷ 2（16bit） ÷ 31,250Hz ÷ 144種 ≈ **約466ms** |
| 音域別サンプル数 | 各マルチサウンドは4〜6音域をカバー（シングルレイヤー） |
| ベロシティレイヤー | なし（単一レイヤー） |
| 1音色あたりのメモリ | 2オシレーター × 28KB ≈ **約56KB**（推定） |

**時代的意義：** Korg M1は1988〜1993年にかけて約25万台を販売し、「史上最も売れたシンセサイザー」の一つ。4MBという当時としては大容量のROMと充実した波形ライブラリが、PCMワークステーション時代の幕を開けた。

### Roland SC-55 Sound Canvas（1991年）

| 項目 | 内容 |
|---|---|
| 総波形ROM容量 | **約3MB**（3チップ × 各1MB） |
| サンプリング周波数 | **32 kHz** |
| ビット深度（格納） | **8.25bit**（8bitマンティッサ + 4bit指数/16サンプルブロック） |
| ビット深度（展開後） | **実効約18bit**（出力時に18bitへ展開） |
| ステレオ/モノラル | 主にモノラル（一部ステレオ） |
| 総波形数 | 推定 **588種前後**（Nuked-SC55プロジェクトの解析による） |
| 総音色数 | **317トーン + 9ドラムキット** |
| 最大ポリフォニー | 24声 |
| データ圧縮 | **Roland独自「8.25bit浮動小数点圧縮」** |

**圧縮方式の詳細：**  
16サンプルブロック単位で、8bitのマンティッサ（仮数部）と4bitの指数部（シフト量）を組み合わせた独自フォーマット。最大シフト量10bitにより実効的なダイナミックレンジは約18bit相当。物理3MBのROMが実効的には数倍の音質情報量を実現している。

| 推定値 | 計算 |
|---|---|
| 1波形平均物理容量 | 3MB ÷ 588 ≈ **約5KB**（圧縮後） |
| **1波形平均長（時間）** | 3MB ÷ 1.03（8.25bit） ÷ 32,000Hz ÷ 588種 ≈ **約155ms** |
| 1音色あたりの波形 | 最大2パーシャル × 最大複数サンプル参照 |

### Yamaha TG500（1992年）

| 項目 | 内容 |
|---|---|
| 総波形ROM容量 | **8MB** |
| サンプリング周波数 | 推定 **44.1 kHz**（AWM2世代） |
| ビット深度 | 混合（16bit/12bit/8bit線形/8bit対数を混合圧縮） |
| ステレオ/モノラル | 主にモノラル |
| 総波形数 | **294種**（Preset 1: 244種 + Preset 2: 50種） |
| 総音色数 | プリセット256 + ユーザー128 + パフォーマンス192 |
| 最大ポリフォニー | 64声 |
| データ圧縮 | **Yamaha AWM2独自圧縮**（混合ビット深度方式） |

| 推定値 | 計算 |
|---|---|
| 1波形平均容量 | 8MB ÷ 294 ≈ **約27KB** |
| **1波形平均長（時間）** | 8MB × 2（圧縮2:1換算） ÷ 2（16bit相当） ÷ 44,100Hz ÷ 294種 ≈ **約617ms** |
| 音域別サンプル数 | ピアノ等で6〜8音域（推定） |

**Yamaha AWM2圧縮の特徴：**  
音声帯域・重要度に応じてビット深度を変える混合方式。高音域（人間の知覚感度が低い帯域）は8bit対数圧縮、中音域は12bit、核心音域は16bitという使い分けにより、「8MBのROMが実質的な音質では数十MB相当に聴こえる」効果を実現。

| 項目 | 内容 |

| ピアノの音域別サンプル数 | 6〜8音域 + ベロシティ2レイヤー = **12〜16サンプル**（推定） |
| ピアノ音色のメモリ | 16 × 18KB ≈ **約290KB**（推定） |

### サンプル長（録音時間）の推定

#### 計算方法

1サンプルの時間長は以下の式で求められる：

```
サンプル長（秒） = 総ROM容量（バイト） ÷ bytes_per_sample ÷ サンプリング周波数（Hz） ÷ 波形数

bytes_per_sample:
  16bit 線形（非圧縮）: 2.0 バイト
  8bit DPCM（Roland JV系）: 1.0 バイト（同じ時間長を半分の容量に）
  8.25bit 浮動小数点（Roland SC系）: 1.03 バイト
  KORG/Yamaha 2:1 圧縮: 1.0 バイト（16bitを2:1で圧縮）
  16bit ステレオ: 4.0 バイト（ch数×2）
```

圧縮はサンプル「数」を減らさず「ビット深度（音質）」を下げる方式がほとんど。  
したがって圧縮してもサンプルの**時間長は変わらない**（データ量は減るが再生時間は同じ）。

#### 楽器カテゴリ別の典型的なサンプル長（推定）

PCMサンプルは「ループあり」と「ループなし（ワンショット）」に大別され、時間長の性質が全く異なる。

**ループあり（持続音系）：**  
アタック部（ループ前）+ ループ部（繰り返し）の2段構成。ストレージに格納されるのはこの合計。  
ループがあることで**格納は数百ミリ秒でも実際は無限に鳴り続ける**。

| 楽器カテゴリ | アタック部（pre-loop） | ループ部（loop） | 合計格納時間 |
|---|---|---|---|
| ピアノ（中音域） | 200〜500ms | 100〜300ms | **300〜800ms** |
| ピアノ（低音域） | 400ms〜1s | 200〜500ms | **600ms〜1.5s** |
| ストリングス | 100〜300ms | 100〜500ms | **200〜800ms** |
| オルガン | 20〜50ms | **20〜100ms** | 40〜150ms |
| ブラス（ホーン） | 100〜200ms | 50〜200ms | **150〜400ms** |
| ウッドウインズ | 100〜250ms | 50〜200ms | **150〜450ms** |
| ヴォーカル/コーラス | 200〜500ms | 200〜500ms | **400ms〜1s** |
| ギター（弦ループ） | 100〜300ms | 100〜300ms | **200〜600ms** |

**ループなし（ワンショット・打楽器系）：**  
自然な減衰を格納する必要があるため、楽器の減衰特性でサンプル長が決まる。

| 楽器カテゴリ | 典型的なサンプル長 | 特記事項 |
|---|---|---|
| バスドラム | **200〜400ms** | 低音の減衰を十分捉えるために比較的長め |
| スネアドラム | **300〜500ms** | スナッピーの余韻まで格納 |
| ハイハット（クローズ） | **100〜200ms** | 減衰が早い |
| ハイハット（オープン） | **500ms〜1s** | 開いた状態の余韻を格納 |
| ライドシンバル | **1〜3s** | 金属の長い余韻が必要 |
| クラッシュシンバル | **2〜5s** | 最も長い。大きなROMコストの原因 |
| トム | **300〜700ms** | サイズにより変動 |
| ウッドブロック/カウベル | **200〜500ms** | 短めの金属音 |
| ピアノ（ハーフペダル） | **2〜6s** | ペダルを踏んだ長い余韻を再現するため長大 |

#### ループ構造と「格納時間」の関係

```
ピアノ A3（440Hz）の例（KORG Triton 相当、推定）：

  格納データ構造:
  ┌───────────────────────┬────────────────┐
  │   アタック部（pre-loop）   │  ループ部（loop）  │
  │   約400ms                │  約200ms         │
  └───────────────────────┴────────────────┘
  ← ←  合計格納時間: 約600ms  → →

  再生時:
  ┌───────────────────────┬────────────────────────────────── …
  │   アタック部（1回再生）   │  ループ部（ループを繰り返し）→ 無限に鳴る …
  └───────────────────────┴────────────────────────────────── …
```

#### D-50の特殊性：「アタック専用PCM」

D-50のLA合成では、PCMサンプルを2種類に分けて使用する：

1. **アタックPCM（47種）**：純粋に「立ち上がりの瞬間」だけを格納した非常に短いサンプル
   - 典型的な長さ：**60〜200ms**（ほとんどはループなし）
   - 目的：デジタルオシレーターが再現できない自然なアタックトランジェントを担当

2. **ループPCM（29種）**：サステイン部分のループのみを格納
   - 典型的な長さ：**100〜500ms**（ループで無限に持続）
   - アタック後に別のPCM波形に切り替えて使用

このアーキテクチャにより、わずか512KBで多彩な音色を実現した。

## 8. 考察・主要な発見

### ROM容量は17年間で約300〜5,000倍に拡大

1987年のRoland D-50（512KB）から2004年のKORG Triton Extreme（160MB）まで、物理ROM容量は約**320倍**。圧縮・展開技術の実効値を加味すると数千倍相当。半導体価格の急速な下落とともに「音色1個あたりのコスト」が劇的に低下した。

### 「実容量」と「実効容量」の乖離

物理ROM容量と音質的な実効容量の間には大きな乖離があった：

- **Roland SC-55**：物理3MBが浮動小数点展開後は実効18bit相当（情報量は数倍以上）
- **Yamaha AWM2（MU50）**：4MBの物理ROMが「数十MB相当」の音質と評価される
- **KORG Trinity**：24MBを2:1圧縮で実効48MB相当に拡大
- **E-mu Proteus**：16bit非圧縮でそのままの容量。「表記通りの容量」だが音質は最高水準

### サンプリング周波数は「独自クロック → 標準規格」へ

1980年代後半の各社は独自クロックを使用していた（D-50: 33.375kHz、M1: 31.25kHz、Proteus: 39kHz）。1990年代後半以降、KORG 48kHz / Yamaha 44.1kHz / Roland 32kHz（後に44.1kHz）という独自規格へと収束し、2000年代には44.1/48kHzが業界標準となった。

### ステレオ化の波

1990年代前半まではほぼ全機種がモノラルサンプルを使用。記憶容量の制約から「ステレオで格納するとモノラルの2倍のROMが必要」という事情があったため。KORG Triton（1999年）でステレオ波形が主流となり、Roland XV-5080（2000年）で「真のステレオ波形」が本格採用された。

### ベロシティレイヤーは「贅沢品」から「標準」へ

- 1988年のM1世代：ベロシティレイヤーはほぼ存在しない（容量的に不可能）
- 1994年のJV-1080世代：2〜4段のベロシティレイヤーが一部音色に登場
- 2000年代：8段以上が標準となり、ピアノ音色では16〜128サンプルに及ぶものも

### 1音色（ピアノ）あたりのメモリ割り当て推移（推定）

| 時代 | 代表機種 | ピアノ音色のメモリ（推定） |
|---|---|---|
| 1988年 | KORG M1 | 約56〜84KB（3音域） |
| 1989年 | E-mu Proteus/1 | 約100〜160KB（5音域） |
| 1994年 | Roland JV-1080 | 約200〜400KB（8音域×2レイヤー） |
| 1999年 | KORG Triton | 約1〜2MB（10音域×4レイヤー・ステレオ） |
| 2004年 | Roland Fantom-X | 約2〜8MB（16音域×8レイヤー・ステレオ） |
